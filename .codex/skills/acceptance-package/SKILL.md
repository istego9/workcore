# Acceptance Package

## Use when
- User asks for an acceptance package.
- User asks to attach screenshots to acceptance/e2e output.
- User asks to produce a reusable handoff bundle for review.

## Inputs to confirm
- `task_id` (artifact id, e.g. `wf-123-acceptance`)
- target URL (default: `http://workcore.build/?e2e=1`)
- optional selector to wait for (default: `body`)

## Standard workflow
1) Run acceptance packaging script:
   - `./scripts/acceptance_package.sh --task-id <task_id> --url <url> --selector "<css>"`
2) For full stack verification, run full suite (acceptance packaging is mandatory there):
   - `ACCEPTANCE_TASK_ID=<task_id> ./scripts/e2e_suite.sh`
3) Verify produced bundle:
   - `artifacts/acceptance/<task_id>/ACCEPTANCE.md`
   - `artifacts/acceptance/<task_id>/screenshots/desktop.png`
   - `artifacts/acceptance/<task_id>/screenshots/mobile.png`
   - `artifacts/acceptance/<task_id>.zip`

## Required report in response
- Absolute path to ZIP.
- What checks were executed.
- If any check was skipped/failed, include exact reason.

## Notes
- Screenshots are generated by Playwright acceptance test (`apps/builder/e2e/acceptance-screens.spec.ts`).
- Do not invent acceptance results; report only executed commands.
